# Чек-лист сдачи

## Что должно быть в GitHub-репозитории

- каталог `k8s/base`
- каталог `k8s/overlays/dev`
- каталог `k8s/overlays/prod`
- каталог `argocd` с `Application`-манифестом(ами)
- документация в `docs` (краткий runbook и подтверждение проверок)

## Минимальные проверки перед сдачей

1. `kustomize build k8s/overlays/dev` отрабатывает без ошибок.
2. `kustomize build k8s/overlays/prod` отрабатывает без ошибок.
3. Все pods в namespace в состоянии `Running/Completed`.
4. `frontend` доступен снаружи кластера (через Ingress, NodePort или LoadBalancer).
5. API-цепочка `frontend -> bff -> services` работает.
6. Загрузка файла/картинки успешна, объект появляется в S3 bucket.
7. Правила `nodeAffinity` реально влияют на placement Pod.
8. Argo CD показывает `Synced/Healthy`.

## Формат сдачи

Передается:

- ссылка на GitHub-репозиторий
- ветка с финальным решением
- краткий `README` в репозитории (как запустить и как проверить)

## Критерии оценки

- **Отлично**: все требования реализованы, автодеплой работает, есть чистая структура `kustomize`, корректный S3 CSI и уверенная защита.
- **Хорошо**: ядро работает, но есть недочеты в production-части (ресурсы/affinity/оформление).
- **Удовлетворительно**: запущен только dev-сценарий, часть требований выполнена частично.
- **Не зачтено**: нет воспроизводимого деплоя, отсутствуют ключевые части (S3 CSI, kustomize overlays, Argo CD).
